Lazy Loading

Lazy Modules in Angular are modules that load only when the user navigates to a specific route, rather than loading with the rest of the application at startup. This approach improves the application's performance by reducing the initial download size

1 Remove the module from AppModule: If the module is being imported directly into AppModule, remove this import.
2 Configure the module with loadChildren in the route configuration. Use a function that loads the module only when its route is requested:
i.e 
  { path: 'route', loadChildren: () => import('path/to/module').then(m => m.ModuleName) }
3 Modify the child route of the module to use an empty path if it's loading as a child of the main route.

Route Guards

Guards in Angular are functions that run before loading a route, allowing or denying access based on specified conditions. Angular provides several types of guards:

1 CanActivate: Controls initial access to a route. For example, it can be used to check if a user is logged in.
2 CanActivateChild: Controls access to child routes of a specific node.
3 CanDeactivate: Allows handling route exits, such as showing a confirmation dialog if there are unsaved changes.
